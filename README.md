# OnePay Bank Integration Service

Серверное приложение для интеграции OnePay с внешними банковскими партнерами.

На текущем этапе сервис реализует взаимодействие OnePay с Kaspi по сценариям:
- получение списка договоров клиента;
- получение информации по погашению;
- расчет суммы для частичного/полного погашения;
- получение параметров ранее созданного платежа;
- прием callback о результате платежа;
- резервная сверка статуса платежа через API банка.

## Назначение

Сервис выделяется в отдельное приложение и отдельный репозиторий для изоляции банковской интеграции от основного приложения Платежного Шлюза.

Основные задачи сервиса:
- прием и обработка запросов Kaspi;
- валидация подписи запросов;
- проверка идемпотентности входящих уведомлений;
- маршрутизация вызовов во внутренние сервисы OnePay;
- фиксация и актуализация статусов платежей;
- регистрация invoice / payment на стороне Kaspi для сценария, стартующего в OnePay / Ломбарде;
- выполнение резервной сверки статуса при отсутствии или ошибке callback.

## Текущий функциональный контур

### Входящие API OnePay для Kaspi
- `GET /api/kaspi/contracts`
- `GET /api/kaspi/repayment`
- `GET /api/kaspi/calculate-repayment`
- `GET /api/kaspi/payment`
- `POST /api/kaspi/callback`

### Исходящие вызовы в Kaspi
- регистрация invoice / payment;
- `GET /api/payment/status`

## Ключевые бизнес-правила
- все запросы должны проходить проверку подписи;
- callback должен обрабатываться идемпотентно;
- статус платежа должен обновляться безопасно для повторных запросов;
- redirect клиента не считается подтверждением факта оплаты;
- источником подтверждения оплаты является callback банка либо резервная сверка статуса.
